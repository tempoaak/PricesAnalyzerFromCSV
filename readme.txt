Представлены консольная утилита для получения выборки 1000 самых дешевых объектов из всех, но не более 20-ти объектов на один ID.
Прогармм для работы использует потоки, т.е. файлы обрабатываются параллельно.
Версия 3
Программа написана на Java 8.
Исходники представлены в виде maven проекта Eclipse
В папке samples находятся несколько файлов csv с тестовыми данными для проверки. (100000 записей в каждом, кроме csv1 и csv2)
Кодировка входных файлов должна быть utf-8
Разделитель дробной части поля Price - точка
Для удобства запуска разных версий, сформированы два jar файла (pricesAnalyzerCsv3.jar) 
Для запуска необходимо использовать команду java -jar pricesAnalyzerCsv3.jar <Необязательный параметр с папкой с входными файлами>
Выходной файл имеет название output.csv (для наглядности сформирован) и создается в текущей папке запуска программы.
Так же в проекте я сделал небольшую утилиту CreateTestBigData, с помощью которой сформировны тестовые файлв в папке samples
В файле result.log представлен консольный лог работы прогарммы на моем компе, с использованием 100 файлов по 100000 записей в каждом (Core i7, 16Гб)
